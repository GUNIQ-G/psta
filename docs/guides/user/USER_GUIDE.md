## 1. 시작하기

### 1.1 최초 설치 (관리자)

시스템이 설치되지 않은 경우 자동으로 설치 마법사(`/install`)로 이동합니다.

**4단계 설치 마법사**:

| 단계 | 내용 |
|------|------|
| 1단계 | DB 연결 확인 (실패 시 다음 단계로 진행 불가) |
| 2단계 | 프론트엔드 URL 설정 (현재 브라우저 주소 자동 입력) |
| 3단계 | 관리자 비밀번호 설정 |
| 4단계 | 설치 실행 |

### 1.2 로그인

**접속 URL**: 설치 시 설정한 서버 주소로 접속 (예: `http://your-server-ip:3000`)

로그인 화면에는 시스템 로고/이름/저작권이 표시됩니다. 로고가 없는 경우 파란 박스의 'P' 문자가 표시됩니다.

**로그인 방법**:
1. Username과 Password 입력
2. **로컬 계정**: 관리자가 생성한 계정으로 즉시 로그인
3. **LDAP 계정**: 사내 계정으로 로그인 → 첫 로그인 시 승인 요청 → 관리자 승인 후 대시보드 이동

**기본 관리자 계정**:
- Username: `admin`
- Password: 설치 마법사에서 설정한 비밀번호 (프로필 페이지에서 변경 가능)

### 1.3 승인 요청 (신규 사용자)

LDAP 또는 첫 로그인 시 자동으로 승인 요청 페이지로 이동합니다. 이 페이지는 두 가지 상태로 표시됩니다.

**승인 요청 전 상태**:
1. 요청 메시지 입력 (선택)
2. "승인 요청" 버튼 클릭
3. 시스템이 관리자에게 알림 전송

**대기 중 상태** (요청 제출 후):
- 요청 시각과 메시지가 화면에 표시됨
- 관리자가 승인하면 자동으로 대시보드로 이동 (페이지를 새로 고칠 필요 없음)

## 2. 메뉴 구조

### 개인 작업
- **대시보드** (`/dashboard`) - 전체 프로젝트/액션 현황 및 팀별 요약
- **작업요청** (`/requests`) - 작업 요청 생성/관리

### 프로젝트 일정
- **일정관리** (`/psta`) - PSTA 트리 뷰 + WBS 타임라인 탭
- **WBS** (`/wbs`) - 전용 간트 차트 페이지
- **보고서** (`/report`) - 스냅샷 보고서 저장/인쇄

### 데이터 관리
- **클라이언트 관리** (`/clients`) - 고객사 CRUD
- **프로젝트 관리** (`/projects`) - 프로젝트 CRUD
- **서비스 관리** (`/services`) - 서비스 CRUD
- **팀별 현황** (`/team-status`) - 팀 기반 액션 현황
- **액션 관리** (`/actions`) - 액션 목록 및 생성
- **통합 파일 관리** (`/integrated-files`) - 전체 파일/링크 통합 조회

### 프로젝트 조직
- **조직도** (`/organization`) - 조직 트리 및 역할 관리

### 시스템 지원
- **메시지** (`/messages`) - 내부 메시지 받은함/보낸함
- **버그/건의** (`/feedback`) - 버그 신고 및 기능 제안

### 시스템 설정 (관리자)
- **멤버 관리** (`/members`) - 사용자 목록 및 LDAP 탭
- **팀 관리** (`/teams`) - 팀 CRUD 및 LDAP 동기화
- **회원 관리** (`/users`) - 레거시 전체 회원 목록
- **사용자 승인** (`/user-approval`) - 승인 대기 목록
- **조직 관리** 내 권한 관리 (`/permissions`) - 역할별 권한 설정
- **일반** (`/general-settings`) - 시스템 전반 설정
- **LDAP 인증** (`/ldap-auth`) - LDAP 서버 설정
- **LDAP 동기화** (`/ldap-sync`) - 수동 동기화
- **LDAP 서버 관리** (`/ldap-admin`) - 서버 직접 접근
- **알림앱 연동** (`/notification-apps`) - 외부 알림 채널 설정

## 3. 주요 기능 사용법

### 3.1 일정관리 (PSTA)

**경로**: 프로젝트 일정 > 일정관리 (`/psta`)

일정관리는 **ItemTree 탭**과 **WBS 타임라인 탭** 두 가지 탭으로 구성됩니다.

#### ItemTree 탭 - 기본 작업 화면

**PSTA 계층 구조**:
```
Project (프로젝트)
  └─ Service (서비스)
       └─ Action (액션) ← 팀이 여기 소속
```

**계층 보기 조절** (P / S / T / A 버튼):
- **P**: 프로젝트만 보기
- **S**: 프로젝트 + 서비스까지 보기
- **T**: 팀까지 보기
- **A**: 전체 보기 (액션까지)
- 각 행의 개별 버튼으로 해당 항목만 펼치기/접기 가능

**필터 사용**:
- "내 업무만 보기": 자신이 속한 팀의 항목만 표시
- "미정 항목 숨기기": 미정 프로젝트/서비스 숨김
- 고객/프로젝트 다중 선택 필터 지원

**항목 선택 시 동작**:
- 오른쪽 상세 패널에 선택 항목 표시
- 트리 패널이 화면의 40%로 자동 축소되어 상세 패널 공간 확보

**딥링크 (URL 파라미터)**:
- `?itemId=<ID>&edit=true` 형식의 URL로 직접 접근하면 해당 항목이 자동으로 열립니다.
- 예: `http://your-server:3000/psta?itemId=abc123&edit=true`

**Excel 내보내기/가져오기**:
- 설정 드롭다운 버튼에서 "Excel 내보내기" / "Excel 가져오기" 선택

#### WBS 타임라인 탭

- 일정관리 화면 내에서 타임라인 뷰로 전환
- 자세한 사용법은 [3.3 WBS / 간트 차트](#33-wbs--간트-차트) 참조

### 3.2 액션 등록 Drawer

사이드바 또는 일정관리 화면에서 "액션 등록" 버튼을 클릭하면 Drawer가 열립니다.

**단계별 입력**:
1. **고객 선택** - 고객사 선택
2. **프로젝트/서비스 선택** - 고객 선택 후 해당 고객의 프로젝트 목록이 자동 로드, 프로젝트 선택 후 서비스 선택
3. **액션 정보 입력**:
   - 제목, 담당자, 시작일/종료일, 진행률
   - 파일 첨부 (여러 파일 동시 선택 가능)
   - URL 링크 추가 (링크 입력 후 포커스 이동 시 페이지 제목 자동 조회)

**Drawer 확장**:
- 우측 상단 확장 버튼으로 Drawer 너비를 넓게 조정 가능

**계층 구조 미완성 시**:
- 선택한 서비스 하위에 팀이 없거나 구조가 불완전한 경우 계층 구조 요청 모달이 표시됩니다.
- 기존 항목 선택 또는 새로운 생성 요청 중 선택

### 3.3 WBS / 간트 차트

**경로**: 프로젝트 일정 > WBS (`/wbs`)

WBS 전용 페이지로, 일정관리 내 타임라인 탭과 동일한 간트 차트를 제공합니다.

**기본 사용**:
1. 상단에서 고객 선택
2. 해당 고객의 프로젝트/서비스/액션 데이터 자동 로드

**날짜 범위 필터**:
- 날짜 범위 직접 선택
- 빠른 버튼: 3개월 / 6개월 / 12개월

**뷰 모드 전환** (6가지):
| 뷰 모드 | 설명 |
|---------|------|
| 주 단위 | 주 기준 헤더 |
| 격주 단위 | 2주 기준 헤더 |
| 월 단위 | 월 기준 헤더 |
| 분기 단위 | 분기 기준 헤더 |
| 연 단위 | 연 기준 헤더 |
| 5년 단위 | 5년 기준 헤더 |

**프로젝트/팀 다중 필터**:
- 프로젝트 드롭다운: 다중 선택으로 특정 프로젝트만 표시
- 팀 드롭다운: 다중 선택으로 특정 팀이 포함된 서비스만 표시

**항목 클릭**:
- 간트 바 또는 행 클릭 시 상단 상세 패널이 슬라이드 인으로 표시

**간트 바 리사이즈**:
- 바 양쪽 핸들을 드래그하여 시작일/종료일 직접 조정 가능

### 3.4 보고서

**경로**: 프로젝트 일정 > 보고서 (`/report`)

**보고서 생성**:
1. 고객 선택
2. 날짜 범위 선택
3. 통계 카드 자동 표시 (전체/완료/진행중 액션 수, 평균 진행률)
4. 프로젝트별 그룹화된 테이블 확인

**스냅샷 저장**:
- "저장" 버튼으로 현재 보고서 상태를 스냅샷으로 저장
- 제목은 "주차 보고서" 형식으로 자동 생성 (예: "2026년 25주차 보고서")
- 저장된 스냅샷 목록에서 과거 보고서 조회 가능

**인쇄**:
1. 보고서 화면에서 "인쇄" 버튼 클릭
2. 브라우저 인쇄 다이얼로그 실행 (인쇄 최적화 CSS 자동 적용)

**스냅샷 삭제**:
- 저장된 스냅샷 목록에서 삭제 버튼 클릭

### 3.6 작업 요청

**경로**: 개인 작업 > 작업요청 (`/requests`)

#### 탭 구성
- **받은 요청**: 내가 받은 작업 요청
- **보낸 요청**: 내가 보낸 작업 요청
- **전체**: 전체 요청 목록

**Segmented 필터** (각 탭 내):
- 전체 / 내 담당 / 팀 담당

#### 새 작업 요청 생성
1. "새 작업 요청" 버튼 클릭
2. 제목, 설명, 우선순위, 마감일 입력
3. "생성" 버튼 클릭

#### 상태 전환 흐름
```
PENDING (대기)
  ├─ 승인 → 승인됨
  ├─ 반려 → 반려됨
  ├─ 협의 → 협의 중 (담당자와 내용 조율)
  └─ 회수 → 회수됨 (요청자가 직접 취소)

협의 중
  ├─ 승인 → 승인됨
  └─ 반려 → 반려됨

반려됨
  └─ 재요청 → PENDING (대기)

승인됨
  ├─ 액션 생성 → ACTION 자동 생성 → 일정관리 편집 페이지로 이동
  └─ 승인 취소 → PENDING (대기)

전달 기능: 개인 간 담당자 재위임 (어느 상태에서든 사용 가능)
```

**계층 구조 미완성 시**: 액션 생성 전 자동으로 계층 구조 요청 모달이 표시됩니다.

**딥링크**: `?workRequestId=<ID>` URL 파라미터로 특정 요청을 자동으로 열 수 있습니다.

### 3.7 액션 관리

**경로**: 데이터 관리 > 액션 관리 (`/actions`)

- ACTION 타입 아이템 전체 목록 표시
- 각 행 왼쪽의 4px 컬러 바로 상태(미시작/진행중/완료/보류) 구분
- 계층 태그(P/S/T)로 소속 프로젝트·서비스·팀 표시
- "액션 생성" 버튼으로 새 액션 등록

---

### 3.8 프로젝트 등록

**경로**: 데이터 관리 > 프로젝트 관리 (`/projects`)

**3단계 위자드 (프로젝트 추가 버튼 클릭)**:
1. 고객 선택
2. 담당자 선택 (기본: 현재 사용자)
3. 프로젝트명, 시작일/종료일, 설명 입력 후 생성

**미정 프로젝트 숨기기**: 상단 체크박스로 "미정 프로젝트" 행 숨김 가능

**프로젝트 삭제 시**: 이동된 팀 수와 액션 수를 확인 메시지로 표시

---

### 3.9 서비스 등록

**경로**: 데이터 관리 > 서비스 관리 (`/services`)

**3단계 위자드 (서비스 추가 버튼 클릭)**:
1. 프로젝트 선택
2. 담당자 선택
3. 서비스명, 시작일/종료일, 설명 입력 후 생성

**통합 미정 서비스**: 목록 상단에 "통합 미정 서비스" 가상 행이 표시됩니다 (미배정 팀/액션의 집합).

### 3.10 클라이언트 관리

**경로**: 데이터 관리 > 클라이언트 관리 (`/clients`)

**클라이언트 추가**:
1. "클라이언트 추가" 버튼 클릭
2. 우측 Drawer에서 정보 입력:
   - 고객명, 코드(unique), 대표자, 사업자번호, 전화번호, 이메일, 주소
   - 로고 이미지 업로드 (5MB 제한)
3. "저장" 버튼 클릭

**Drawer 확장**:
- 우측 상단 확장 버튼으로 Drawer 너비를 `50%` → 화면 전체(`calc(100vw - 200px)`)로 전환

**클라이언트 상세/수정**:
1. 테이블에서 행 클릭 → 상세 모달 열림 (읽기 모드)
2. 상세 모달에서 "수정" 버튼 클릭 → 수정 모드 전환
3. 정보 수정 후 "저장"
4. 연결된 프로젝트 목록도 상세 모달에서 확인 가능

### 3.11 팀별 현황

### 3.12 통합 파일 관리

**경로**: 데이터 관리 > 통합 파일 관리 (`/integrated-files`)

파일과 URL 링크를 단일 화면에서 통합 조회합니다.

**주요 기능**:
- PSTA 위치 태그(P/S/T/A)로 각 파일/링크의 계층 위치 표시
- 고유 값 기반 컬럼 자동 필터 (파일 유형, 확장자, 업로더 등)

- 링크는 새 탭으로 열기

### 3.13 파일 첨부

**액션 생성 시 파일 첨부**:
1. 액션 등록 Drawer에서 3단계(액션 정보) 진행
2. "파일 첨부" 섹션에서 파일 선택 (다중 선택 가능)
3. URL 링크 추가 시 페이지 제목 자동 조회

**기존 액션에 파일 추가**:
1. 일정관리에서 액션 클릭 → 상세 모달
2. "첨부 파일" 탭 또는 섹션에서 "파일 업로드" 클릭
3. 파일 선택 → 즉시 업로드

**파일 다운로드/삭제**:
- 다운로드: 파일 목록에서 다운로드 아이콘 클릭
- 삭제: 삭제 아이콘 클릭 (ADMIN 또는 업로더만 가능)

### 3.14 메시지

**경로**: 시스템 지원 > 메시지 (`/messages`)

- **받은함 / 보낸함** 탭으로 구분
- 상단 네비게이션 바에 읽지 않은 메시지 수 배지 표시
- 30초마다 자동 새로고침 (폴링)
- 메시지 내 항목 링크([ITEM_INFO] 태그) 클릭 시 해당 페이지로 이동

### 3.15 알림 센터

상단 네비게이션 바의 벨 아이콘으로 인앱 알림을 확인합니다.

- 읽지 않은 알림 수 배지 표시
- 알림 클릭 시 해당 페이지로 이동 (타입별 이동 경로 자동 결정)
- "전체 읽음" 버튼으로 모든 알림 일괄 처리

### 3.16 버그/건의 (피드백)

**경로**: 시스템 지원 > 버그/건의 (`/feedback`)

**새 피드백 작성**:
1. "피드백 작성" 버튼 클릭
2. 폼 입력:
   - 제목 (필수)
   - 유형 선택: **버그 보고 / 기능 개선 / 제안**

3. "등록" 버튼 클릭

**상태별 통계 카드**: 대기/검토중/완료/반려 건수 확인

**수정 가능 조건**: 본인이 작성하고 아직 "대기" 상태인 항목만 수정 가능

**관리자 기능**: 상태 변경(검토중/완료/반려) 및 코멘트 추가

**에디터 단축키**:
| 단축키 | 기능 |
|--------|------|
| Ctrl+B | 굵게 |
| Ctrl+I | 기울임 |
| Ctrl+Z | 실행 취소 |
| Ctrl+Y | 다시 실행 |
| Ctrl+V | 클립보드 이미지 붙여넣기 |

### 4.8 LDAP 인증 설정

**경로**: 시스템 설정 > LDAP 인증 (`/ldap-auth`)

LDAP 서버를 다중으로 등록하고 관리합니다.

**서버 목록**:
- 각 서버의 마지막 테스트 결과와 상대 시간 표시 ('방금 전', 'N시간 전', 'N일 전')

**서버 상세 설정** (`/ldap-settings/:id`):
- OpenLDAP / Active Directory 프리셋 템플릿 선택으로 빠른 설정
- 연결 상태: 초록(성공) / 빨강(실패) 컬러 카드
- "연결 테스트" 버튼으로 즉시 확인 (응답 시간 측정)
- 고급 섹션(접기/펼치기): 검색 필터, 속성 매핑
- DN 형식 검증기 내장

**계층형 미리보기 모달**:
1. "미리보기" 버튼 클릭
2. LDAP 조직 트리를 체크박스로 선택
3. 가상 스크롤 지원 (대용량 디렉토리 대응)
4. 검색으로 특정 항목 필터링
5. "선택 항목만 적용" 버튼으로 부분 동기화 가능

---

### 4.9 LDAP 동기화

**경로**: 시스템 설정 > LDAP 동기화 (`/ldap-sync`)

**Dry-run 모드 (권장 순서)**:
1. "Dry-run 실행" 버튼 클릭 → 실제 변경 없이 결과 미리보기
2. 변경 예정 내용 확인 (추가/수정/비활성화될 사용자 수)
3. 내용 확인 후 "실제 동기화 실행" 버튼 클릭

**동기화 통계 표시**:
- LDAP 그룹 수 / LDAP 사용자 수
- PSTA 팀 수
- 활성 사용자 수 / 비활성 사용자 수
- 마지막 동기화 시간 (상대 시간 표시)

## 5. 휴지통 (소프트 삭제)

PSTA에서 항목 삭제 시 즉시 제거되지 않고 휴지통으로 이동합니다.

**휴지통 특징**:
- 삭제 후 **30일 이내** 복원 가능
- 30일 경과 시 자동 영구 삭제
- 통계 표시: 삭제된 항목 수 및 만료 예정 항목 수

**복원 방법**:
1. 일정관리 또는 관련 관리 페이지에서 "휴지통" 버튼 클릭
2. 삭제된 항목 목록 확인
3. 복원할 항목 선택 후 "복원" 버튼 클릭

**영구 삭제**:
- 휴지통에서 개별 영구 삭제 또는 일괄 영구 삭제 가능
- ADMIN만 다른 사용자 항목 영구 삭제 가능

**역할별 권한**:
- **ADMIN**: 모든 항목 복원/영구삭제 가능
- **PO/PM**: 자신이 속한 프로젝트의 항목만 복원 가능
- **MEMBER**: 자신이 생성한 항목만 복원 가능

## 6. 권한별 기능 요약

### ADMIN (최고 관리자)
- 모든 페이지 및 기능 접근 가능
- 다른 사용자의 모든 항목 관리 가능
- 시스템 설정 전체 제어

### PO (프로젝트 책임자)
- 개인 작업, 프로젝트 일정, 데이터 관리 전체 기능
- 조직 관리, 시스템 설정 접근 불가

### PM (프로젝트 관리자)
- 개인 작업, 프로젝트 일정 전체 기능
- 클라이언트/프로젝트 읽기 전용
- 조직 관리, 시스템 설정 접근 불가

### MEMBER (일반 사용자)
- 대시보드, 작업요청 전체 기능
- 일정관리(PSTA) 읽기 전용
- 피드백 작성 가능
- WBS, 보고서, 데이터 관리, 조직 관리, 시스템 설정 접근 불가

> 실제 권한은 관리자가 권한 관리 페이지(`/permissions`)에서 역할별로 커스터마이즈할 수 있습니다.
