## 1. 시작하기

### 1.1 로그인
**접속 URL**: 설치 시 설정한 서버 주소로 접속 (예: http://your-server-ip:3000)

**로그인 방법**:
1. Username과 Password 입력
2. **로컬 계정**: admin이 생성한 계정으로 즉시 로그인
3. **LDAP 계정**: 사내 계정으로 로그인 → 첫 로그인 시 승인 요청 → 관리자 승인 후 대시보드 이동

**기본 관리자 계정**:
- Username: `admin`
- Password: 설치 마법사에서 설정한 비밀번호 (v1.1.32부터 DB에서 관리, 프로필 페이지에서 변경 가능)

### 1.2 승인 요청 (신규 사용자)
첫 로그인 시 자동으로 승인 요청 페이지로 이동:
1. 선택적으로 승인 요청 메시지 작성
2. "승인 요청" 버튼 클릭
3. 관리자 승인 대기
4. 승인 후 → 대시보드 접근 가능

## 3. 주요 기능 사용법

### 3.1 일정관리 (PSTA)

**경로**: 프로젝트 일정 > 일정관리

**계층 보기 조절**:
- **P, S, A 버튼** 클릭으로 계층 수준 조정
  - **P**: 프로젝트만 보기
  - **S**: 프로젝트 + 서비스까지 보기
  - **A**: 전체 보기 (액션까지)

**필터 사용**:
1. 필터 드롭다운 클릭
2. "내 업무만 보기" 체크 → 자신이 속한 팀의 항목만 표시
3. "미정 항목 숨기기" 체크 → 미정 프로젝트/서비스 숨김

**항목 생성**:
1. 부모 항목 선택
2. "+" 버튼 클릭
3. 폼 입력 후 저장

**Excel 내보내기**:
1. "내보내기" 버튼 클릭
2. Excel 파일 다운로드

**Excel 가져오기**:
1. "가져오기" 버튼 클릭
2. Excel 파일 선택
3. 자동으로 데이터 임포트 (중복 방지)

### 3.2 프로젝트 등록

**경로**: 데이터 관리 > 프로젝트 관리 > 프로젝트 등록

**3단계 위자드**:
1. **고객 선택**: 프로젝트가 속할 고객 선택
2. **담당자 선택**: 자동으로 현재 사용자 선택됨 (변경 가능)
3. **프로젝트 정보**:
   - 프로젝트명 입력
   - 시작일/종료일 선택
   - 설명 입력
   - "생성" 버튼 클릭

### 3.3 작업 요청

**경로**: 개인 작업 > 작업요청

**새 작업 요청 생성**:
1. "새 작업 요청" 버튼 클릭
2. 폼 입력:
   - 제목 (필수)
   - 설명 (필수)
   - 우선순위 선택
   - 마감일 설정
3. "생성" 버튼 클릭

**작업 요청 → 액션 변환**:
1. 작업 요청 목록에서 항목 클릭
2. 상세 모달 열림
3. "액션 생성" 버튼 클릭
4. 자동으로 PSTA의 해당 팀 하위에 액션 생성
5. 액션 수정 페이지로 이동 → 상세 정보 입력

**작업 요청 회수**:
1. 상세 모달에서 "회수" 버튼 클릭
2. 확인 → 상태가 "회수됨"으로 변경

### 3.4 파일 첨부

**액션 생성 시**:
1. 액션 등록 Drawer 열기
2. Step 3 (액션 정보) 하단 "파일 첨부" 섹션
3. 파일 선택 (최대 10개)
4. "생성" 버튼 → 액션 생성 후 파일 자동 업로드

**기존 액션에 파일 추가**:
1. 일정관리에서 액션 클릭
2. 수정 모달 열림
3. "첨부 파일" 섹션에서 "파일 업로드" 클릭
4. 파일 선택 → 즉시 업로드

**파일 다운로드/삭제**:
- 다운로드: 파일 목록에서 다운로드 아이콘 클릭
- 삭제: 삭제 아이콘 클릭 (ADMIN 또는 업로더만 가능)

### 3.5 클라이언트 관리

**경로**: 데이터 관리 > 클라이언트 관리

**클라이언트 추가**:
1. "클라이언트 추가" 버튼 (보라색)
2. Drawer에서 정보 입력:
   - 고객명 (필수)
   - 코드 (필수, unique)
   - 대표자, 사업자번호, 전화번호, 이메일, 주소
   - 로고 이미지 업로드 (5MB 제한)
3. "저장" 버튼

**클라이언트 상세보기**:
1. 테이블에서 행 클릭
2. 상세 모달 열림 (읽기 모드)
3. 로고, 기본 정보, 프로젝트 목록 표시

**클라이언트 수정**:
1. 상세 모달에서 "수정" 버튼 클릭
2. 수정 모드로 전환
3. 정보 수정 후 "저장" 버튼

**클라이언트 삭제**:
1. 상세 모달에서 "삭제" 버튼 클릭
2. 확인 → 삭제 (권한 필요)

### 3.6 WBS (Gantt 차트)

**경로**: 프로젝트 일정 > WBS

**날짜 범위 필터**:
- RangePicker로 시작일/종료일 직접 선택
- 빠른 필터: 3개월, 6개월, 1년, 5년, 10년
- 기본값: 오늘 기준 앞뒤 1.5개월

**프로젝트 필터**:
1. 프로젝트 드롭다운 클릭
2. 다중 선택 가능
3. 선택한 프로젝트만 차트에 표시

**팀 필터**:
1. 팀 드롭다운 클릭
2. 다중 선택 가능
3. 선택한 팀이 포함된 서비스만 표시

### 3.7 보고서 작성

**경로**: 프로젝트 일정 > 보고서 작성

**보고서 생성**:
1. 고객 선택
2. 날짜 범위 선택 (기본: 오늘 기준 앞뒤 7일)
3. 통계 카드 자동 표시:
   - 전체 업무 수
   - 완료된 업무 수
   - 진행중인 업무 수
   - 평균 진행률
4. 프로젝트별 그룹화된 테이블 확인

**인쇄**:
1. 보고서 생성 후
2. "인쇄" 버튼 클릭
3. 브라우저 인쇄 다이얼로그

### 3.8 버그/건의

**경로**: 시스템 지원 > 버그/건의

**새 피드백 작성**:
1. "피드백 작성" 버튼 클릭
2. 폼 입력:
   - 제목 (필수)
   - 유형 선택 (버그 보고, 기능 제안, 문의사항 등)
   - 내용 작성 (리치 텍스트 에디터)
3. "등록" 버튼 클릭

**이미지 첨부 방법**:
| 방법 | 설명 |
|------|------|
| Ctrl+V | 클립보드 이미지 붙여넣기 |
| 드래그앤드롭 | 파일을 에디터 영역으로 드래그 |
| 이미지 버튼 | 툴바의 이미지 아이콘 클릭 후 파일 선택 |

**에디터 서식**:
- **Ctrl+B**: 굵게
- **Ctrl+I**: 기울임
- **Ctrl+Z**: 실행 취소
- **Ctrl+Y**: 다시 실행

**팁**: 문제 발생 위치를 경로로 설명하면 더 빠른 해결이 가능합니다.
- 예: "일정관리 > 서비스 트리에서 항목 클릭 > 상세정보 모달의 관련문서 탭"

## 4. 권한별 기능

### 4.1 ADMIN (최고 관리자)
**접근 가능한 모든 페이지**:
- ✅ 개인 작업 (대시보드, 작업요청)
- ✅ 프로젝트 일정 (PSTA, WBS, 보고서)
- ✅ 데이터 관리 (클라이언트, 프로젝트, 서비스, 액션) - 모든 CRUD
- ✅ 조직 관리 (팀, 회원, 사용자 승인) - 모든 CRUD
- ✅ 시스템 설정 (LDAP, 알림앱, 권한) - 모든 CRUD

### 4.2 PO (프로젝트 책임자)
**제한 사항**:
- ❌ 조직 관리 (팀, 회원, 사용자 승인) - 접근 불가
- ❌ 시스템 설정 - 접근 불가

**나머지는 ADMIN과 동일**

### 4.3 PM (프로젝트 관리자)
**제한 사항**:
- ❌ 조직 관리 - 접근 불가
- ❌ 시스템 설정 - 접근 불가
- 📖 클라이언트/프로젝트 - **읽기 전용**

**나머지는 PO와 유사**

### 4.4 MEMBER (일반 사용자)
**접근 가능**:
- ✅ 대시보드 - 전체 기능
- ✅ 작업요청 - 전체 기능
- 📖 일정관리 (PSTA) - **읽기 전용**

**접근 불가**:
- ❌ WBS, 보고서
- ❌ 데이터 관리
- ❌ 조직 관리
- ❌ 시스템 설정
