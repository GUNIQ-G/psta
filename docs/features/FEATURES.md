# PSTA 기능 소개서

**버전**: v1.1.5
**작성일**: 2025-11-12
**대상**: 관리자, 의사결정자, 신규 사용자

---

## 🎯 PSTA란?

**Project-Service-Team-Action** 계층 구조 기반의
**통합 프로젝트 관리 시스템**

PSTA는 프로젝트를 4단계 계층으로 관리하여
복잡한 프로젝트도 명확하게 추적할 수 있는 시스템입니다.

---

## ✨ 주요 기능

### 1. 계층적 프로젝트 관리

**4단계 계층 구조**:
```
Project (프로젝트)
  └─ Service (서비스)
      └─ Team (팀)
          └─ Action (액션)
```

**특징**:
- ✅ Notion 스타일 트리 뷰
- ✅ 상태/진행률 자동 산정
  - TEAM/SERVICE/PROJECT는 하위 항목 기반으로 자동 계산
  - ACTION만 수동 입력
- ✅ 컬러 코딩 (P=보라, S=파랑, T=녹색, A=주황)
- ✅ **아이템 이동 기능** ⭐ NEW
  - 액션/팀/서비스를 다른 부모로 이동
  - 계층 구조 검증 (ACTION→TEAM, TEAM→SERVICE, SERVICE→PROJECT)
  - 이동 후 진행률 자동 재계산
- ✅ 드래그 앤 드롭 (향후)

---

### 2. 작업 요청 시스템

**팀원 → 팀장 작업 요청 워크플로우**:

1. 팀원이 작업 요청 생성
2. 팀장이 승인/거부
3. 승인 시 → 액션으로 자동 변환
4. 양방향 링크로 추적 가능

**특징**:
- ✅ 우선순위 설정 (낮음/보통/높음/긴급)
- ✅ 마감일 설정
- ✅ 요청 회수 기능
- ✅ 작업 요청 ↔ 액션 연결

### 4. LDAP 인증 및 조직 관리

**기존 사내 계정 연동**:
- ✅ LDAP 서버 통합
- ✅ 비밀번호 AES-256-CBC 암호화
- ✅ JWT 토큰 발급
- ✅ 관리자 승인 워크플로우

**LDAP 그룹 → 팀 자동 할당**:
1. LDAP 그룹 정보 조회
2. PSTA 팀과 자동 매핑
3. 로그인 시마다 팀 정보 업데이트

**사용자 승인 절차**:
1. 신규 사용자 첫 로그인
2. 자동으로 승인 요청 페이지
3. 관리자가 승인/거부
4. 승인 후 → 시스템 접근 가능

**조직 관리 통합 페이지** ⭐ NEW (v1.1.5):
- ✅ **트리 구조 조직도**
  - 팀과 사용자를 하나의 트리로 통합 표시
  - 팀 확장 시 소속 사용자 목록 표시
  - 실시간 검색 기능
- ✅ **LDAP 동기화**
  - LDAP를 단일 진실 공급원(Single Source of Truth)으로 사용
  - 수동/자동(매일 02:00 KST) 동기화
  - Dry-run 모드 지원
  - 동기화 이력 및 통계
- ✅ **통합 대시보드**
  - 활성 팀/사용자 통계
  - LDAP 연동 현황
  - 심플한 더존테크윌 디자인
- ✅ **권한 관리**
  - ADMIN만 LDAP 동기화 가능
  - 읽기 전용 인터페이스 (LDAP 기반 관리)

### 5. WBS 및 Gantt 차트

**시각적 일정 관리**:
- 📊 Gantt 차트 뷰
- 📅 날짜 범위 필터 (3개월, 6개월, 1년, 5년, 10년)
- 🔍 **고객/프로젝트 다중 선택 필터** ⭐ NEW (v1.1.4)
  - 체크박스 방식으로 여러 항목 동시 선택
  - 선택 개수 표시 (예: `고객 선택 (2)`)
  - 계층 구조 기반 필터링
- 🎨 계층별 색상 구분

**상세보기 기능** (v1.1.3 신규):
- ✅ **댓글 시스템**
  - 사용자 멘션 (@username)
  - 이모지 지원
  - 실시간 댓글 추가/삭제
- ✅ **관련 문서**
  - 현재 항목 + 하위 항목 파일/링크 계층 표시
  - 파일 다운로드, 링크 새 탭 열기

- ✅ **하위 항목 리스트**

- ✅ **작업 요청 연동**
  - ACTION 타입에 연결된 WorkRequest 표시
  - 클릭으로 작업 요청 페이지 이동

**UI 최적화** (v1.1.3):
- ✅ 페이지 제목/설명 제거로 공간 확보
- ✅ PSTA 업무명 열 너비 증가 (250px → 350px → 450px)

### 8. 권한 관리

**4단계 역할**:
1. **ADMIN** (최고 관리자): 모든 권한
2. **PO** (프로젝트 책임자): 조직 관리/시스템 설정 제외
3. **PM** (프로젝트 관리자): PO + 클라이언트/프로젝트 읽기 전용
4. **MEMBER** (일반 사용자): 대시보드/작업요청 전체, PSTA 읽기 전용

**15개 리소스**:
- 개인 작업: dashboard, requests
- 프로젝트 일정: psta, wbs, report
- 데이터 관리: clients, projects, services, actions
- 조직 관리: teams, users, organization, user-approval
- 시스템 설정: ldap-auth, ldap-sync, permissions

**페이지별 CRUD 권한**:
- ✅ View (보기)
- ✅ Create (생성)
- ✅ Update (수정)
- ✅ Delete (삭제)

**자동 메뉴 필터링**:
- 권한 없는 메뉴는 자동 숨김
- 권한 없는 페이지 접근 시 403 에러
